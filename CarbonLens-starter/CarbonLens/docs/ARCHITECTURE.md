# CarbonLens - Master Architecture Reference

This document contains ALL architecture decisions, database schemas, output schemas, and agent prompts. Claude Code should reference specific sections of this document per task - NOT try to implement everything at once.

---

## System Overview

CarbonLens is a multi-agent AI platform with two modes:
- **Verify Mode**: 5 agents that cross-reference corporate sustainability claims against public data
- **Measure Mode**: 4 agents that calculate Scope 3 emissions from procurement data

**Key design principle**: LLMs handle UNDERSTANDING (parsing, classifying, reasoning). Code handles MATH (lookups, calculations, aggregation). No emission number is ever generated by an LLM.

## Tech Stack
- Backend: Python 3.11+, FastAPI
- Frontend: React + Vite
- AI: Claude API (Sonnet for most agents, Opus for cross-referencing)
- Agent Orchestration: LangGraph (with plain-function fallback)
- Database: SQLite
- Deployment: Google Cloud Run

```python
    "Financial Services": {"scope1_2_tco2e_per_m_revenue": 5, "scope3_multiplier": 12.0, "source": "EPA EEIO"},
    "Food & Beverage": {"scope1_2_tco2e_per_m_revenue": 45, "scope3_multiplier": 5.5, "source": "EPA EEIO"},
    "Chemicals": {"scope1_2_tco2e_per_m_revenue": 200, "scope3_multiplier": 2.0, "source": "EPA"},
}

def estimate_emissions(revenue_usd: float, sector: str) -> dict:
    intensities = SECTOR_INTENSITIES.get(sector)
    if not intensities:
        return {"error": f"No benchmark for sector: {sector}"}
    revenue_m = revenue_usd / 1_000_000
    scope1_2 = revenue_m * intensities["scope1_2_tco2e_per_m_revenue"]
    scope3 = scope1_2 * intensities["scope3_multiplier"]
    return {
        "estimated_scope1_2_mtco2e": round(scope1_2, 1),
        "estimated_scope3_mtco2e": round(scope3, 1),
        "estimated_total_mtco2e": round(scope1_2 + scope3, 1),
        "method": "Revenue-based EEIO estimation",
        "confidence": "low",
        "source": intensities["source"]
    }
```

## VERIFY MODE AGENT PIPELINE

Sequential pipeline: Agent 1 → Agent 2 → Agent 3 → Agent 4 → Agent 5

Each agent is a Python function: takes state dict, returns updated state dict.

```python
def run_verify_pipeline(company_name: str) -> dict:
    state = {"company_name": company_name}
    state = company_intelligence_agent(state)    # Agent 1
    state = report_extraction_agent(state)       # Agent 2
    state = independent_data_agent(state)        # Agent 3
    state = cross_reference_agent(state)         # Agent 4
    state = report_generation_agent(state)       # Agent 5
    return state
```

### Agent 1: Company Intelligence Agent
- LLM: Claude Sonnet
- Tools: web search, local GHGRP SQLite query
- Input: state.company_name
- Output: state.company_profile
- Prompt: see docs/agent_prompts.md

### Agent 2: Report Extraction Agent
- LLM: Claude Sonnet
- Tools: web search (to find PDF), PDF download/parse
- Input: state.company_profile
- Output: state.claims_extracted
- Prompt: see docs/agent_prompts.md

### Agent 3: Independent Data Agent
- LLM: Claude Sonnet with tool use
- Tools: local GHGRP SQLite query, local benchmarks query, estimation engine (Python function), web search
- Input: state.company_profile, state.claims_extracted
- Output: state.independent_data
- NOTE: GHGRP queries are LOCAL SQLite, not live API calls. News/ECHO searches use web search.

### Agent 4: Cross-Reference & Scoring Agent
- LLM: Claude OPUS (reasoning-intensive)
- Tools: none (pure reasoning over accumulated state)
- Input: state.company_profile, state.claims_extracted, state.independent_data
- Output: state.cross_reference_analysis (findings + transparency_score)

### Agent 5: Report Generation Agent
- LLM: Claude Sonnet
- Tools: none (narrative generation)
- Input: all of state
- Output: state.final_report

## MEASURE MODE AGENT PIPELINE

Sequential: Agent 1 → Agent 2 → Agent 3 (CODE) → Agent 4

```python
def run_measure_pipeline(file_content: str) -> dict:
    state = {"raw_input": file_content}
    state = data_ingestion_agent(state)          # Agent 1 (LLM)
    state = category_classification_agent(state)  # Agent 2 (LLM)
    state = emission_factor_calculate(state)      # Agent 3 (CODE - no LLM)
    state = analysis_recommendation_agent(state)  # Agent 4 (LLM)
    return state
```

### Measure Agent 1: Data Ingestion
- LLM: Claude Sonnet
- Input: raw CSV/Excel text
- Output: state.normalized_procurement (clean JSON array)

### Measure Agent 2: Category Classification
- LLM: Claude Sonnet
- Input: state.normalized_procurement
- Output: state.classified_items (enriched with scope3_category + ef_lookup_key)

### Measure Agent 3: Emission Factor Lookup & Calculate
- NO LLM - pure Python code
- Queries SQLite for emission factors
- Applies formulas: emissions = amount × factor
- Output: state.calculated_items

### Measure Agent 4: Analysis & Recommendations
- LLM: Claude Sonnet
- Input: state.calculated_items + pre-computed aggregations
- Output: state.analysis (hotspots, recommendations, data quality)

## FASTAPI ENDPOINTS

```
POST /api/verify          - Start Verify analysis. Body: {company_name: string}. Returns: {job_id}
GET  /api/verify/{job_id} - Poll status. Returns: {status, current_agent, agents: [{name, status, message}], result: null | full_result}

POST /api/measure         - Start Measure analysis. Body: multipart file upload. Returns: {job_id}
GET  /api/measure/{job_id} - Poll status. Same pattern as verify.
```

Jobs run in background threads. Frontend polls every 1-2 seconds.
